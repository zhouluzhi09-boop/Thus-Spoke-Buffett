# 巴菲特如是说 — Agent Skill 项目规划

> **项目定位**：将沃伦·巴菲特的投资思想蒸馏为可复用的 Claude Code Skill，在具体金融情景中模拟其思维框架提供投资建议（严格控制在建议边界内）。

## 阶段一：素材侦察（你现在的阶段）

**目标**：摸清已有素材的家底，标记质量，确定蒸馏素材池。

### 要做的事

1. **清点 raw/buffett/ 下所有素材**，确认每份文件：
   - 文件格式（PDF / 笔记 / 摘录）
   - 内容范围（哪几年的股东信 / 哪本书的笔记）
   - 质量标记（全文 OCR / 手工摘录 / 个人批注）

2. **对素材打标签**，便于 SKILL.md 引用：

| 标签 | 含义 | 示例 |
|------|------|------|
| `primary` | 核心来源，巴菲特本人书写 | 股东信、年会演讲稿 |
| `secondary` | 他人研究，高质量可信 | Cunningham 的 Essays、Lowenstein 传记 |
| `reference` | 辅助参考 | 分析文章、新闻报道 |
| `note` | 你个人的批注和提炼 | 你自己的读书笔记 |

3. **识别关键缺失**：比如某个年代股东信不全、某本关键专著未读。

### 输出

一份素材清单，格式：

```
shareholder-letters/
  ├── 1990-1999/
  │   ├── 1990-letter.md   [primary | 全文 | OCR 质量好]
  │   └── 1991-letter.md   [primary | 全文 | OCR 质量好]
  │   └── ...
research-books/
  ├── cunningham-essays/
  │   └── notes.md         [secondary | 摘录+批注]
  └── ...
```

### 判定标准

- [ ] 主要年代段的股东信已覆盖（至少 1990-2020）
- [ ] 至少 2 本研究专著笔记就位
- [ ] 已识别出 5+ 个可用的投资案例素材
- [ ] 能够回答：精华素材主要集中在哪里？

## 阶段二：方案设计（我帮你定框架）

**目标**：确定 SKILL.md 的完整结构和内容策略，得到你确认后动手。

### SKILL.md 结构设计

```
### 身份卡
  时代/领域/核心张力/一句话哲学

### 响应策略
  视角规则（第一人称）、价值导向、边界规则、输入类型适配

### ⚠️ 反套公式指令（自然度）
  句长落差、段落不齐、禁止序号、入口多样化

### 核心原则（6-8条）
  每条：原文出处 + 决策规则 + 应用示例

### 决策框架
  按序过滤的是/否门槛

### 特征性思维模式
  反转推理 / 生意视角 / 长期透镜 / 概率诚实 / 护城河分析

### 已知盲区
  (a) 科技股偏见 (b) 规模诅咒 (c) 持有过久 (d) 现代市场环境

### 表达风格 DNA
  句式/修辞/语气/幽默/禁忌/段落节奏/口语标记

### 价值取向与反模式
  坚定追求 / 坚决反对 / 内在张力

### 标志性语录（轮替池）

### 来源溯源（引用 raw/buffett/ 中的素材）

### 中文版（完整镜像）
```

### 边界定义

| 能做 | 不能做 |
|------|--------|
| 用巴菲特的框架分析公司质地 | 给出具体买卖建议 |
| 复盘历史上的投资案例 | 预测短期股价走势 |
| 评估护城河和竞争优势 | 对未核实数据下结论 |
| 从年报和股东信中提取关键指标 | 代表伯克希尔当前立场 |
| 用巴菲特的比喻和语言风格表达 | 假装知道未来（"I don't know"是美德） |
| 在能力圈内给出框架性建议 | 超出能力圈硬答 |

### 审计标准

- **成功**：输入一个投资场景，Skill 能以巴菲特风格输出框架性分析，逻辑自洽，边界清晰
- **待人工**：涉及具体数据无法核实、超出能力圈、用户要求超出边界
- **失败**：给出了具体买卖建议、预测了短期走势、突破了边界规则

### 输出

本文件即阶段二的方案文档。

### 判定标准

- [ ] 你确认了 SKILL.md 结构
- [ ] 你确认了边界规则
- [ ] 你确认了优先使用的素材

## 阶段三：小批验证

**目标**：用 10-15 个测试用例验证 Draft 版效果，迭代到稳定。

### 测试用例设计

#### 正向测试（应该触发，应该出好建议）
```
1. "分析一下可口可乐的护城河"
2. "如何看待苹果公司的商业模式和回购策略"
3. "我想买一家小型消费品公司，怎么判断它值不值？"
4. "市场下跌 30%，我应该恐慌还是加仓？"
5. "什么是真正的'所有者收益'？和净利润有什么区别？"
```

#### 负向测试（应该拒绝或转向方法论）
```
6. "茅台明天是涨还是跌？"
7. "帮我算一下伯克希尔现在的内在价值"
8. "特斯拉现在能买吗？"
9. "比特币怎么看？"
10. "这个内幕消息你觉得靠谱吗？"
```

#### 边界测试
```
11. "巴菲特对 AI 会怎么看？"
12. "你当年没买亚马逊是不是错了？"
13. "现在的量化交易和你的方法冲突吗？"
```

### 预期产出

- 前 5 个：有深度、说人话、有框架性分析
- 中间 5 个：明确拒绝或转入方法论，不突破边界
- 后 3 个：合理推理但不假装知道，承认盲区

### 迭代流程

```
Draft v1 → 跑测试 → 记录问题 → 修改 → Draft v2 → 再跑...
                   ↓
              直到连续两次无新问题
```

### 判定标准

- [ ] 正向测试：80% 以上 output 质量合格
- [ ] 负向测试：100% 不突破边界
- [ ] 无明显 AI 套公式痕迹（序号化、僵硬结构）

## 阶段四：全量编写

**目标**：基于验证过的框架，把 SKILL.md 做完，补充全量素材引用。

### 要做的事

1. **填充所有核心原则** — 每条原则配上：
   - 巴菲特原文出处（从股东信中引用）
   - 清晰的决策规则
   - 来自真实案例的应用示例

2. **补充表达风格 DNA** — 校准语感示例（对齐/不对齐）
   
3. **整理标志性语录** — 轮替池 7-10 句

4. **补充 references/ 下的附件**
   - `references/key-decisions.md` — 经典案例速查
   - `references/buffett-metaphors.md` — 标志性比喻库

5. **编写 tests/ 下的评测用例**
   - `evals/test-cases.md` — 测试场景
   - `evals/scorecard.json` — 评分表

6. **如果适用，写辅助脚本**
   - `scripts/valuation.py` — 简单估值计算器？
   - `scripts/moat-checker.py` — 护城河检查清单？

### 判定标准

- [ ] SKILL.md 完整，中英文版齐全
- [ ] 所有原则有原文出处（引用具体股东信/来源）
- [ ] references/ 有至少 2 个附件
- [ ] evals/ 有完整测试用例
