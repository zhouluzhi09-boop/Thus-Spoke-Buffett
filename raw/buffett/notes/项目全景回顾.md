# 巴菲特如是说：项目全景回顾

> 从一份参考资料集合到一个可安装、可校验、可迭代的 AI Skill。

---

## 一、需求规划（2026-06-14）

### 1.1 初始动机

用户收集了 67 份巴菲特原始资料（PDF/EPUB/访谈逐字稿/分析文章），散落在桌面和 Clippings 里。目标不是做资料归档，而是做成一个**可以直接被 AI 调用的决策引擎**——问"巴菲特会怎么看这件事"，能得到基于他真实认知框架的回答，而不是模型的泛泛而谈。

### 1.2 参考系

两个标杆影响了项目方向：

- **Karpathy LLM Wiki**：三层知识库（raw/ 不可变 → wiki/ LLM 维护 → CLAUDE.md 模式文件），定义了"AI 维护知识库"的范式
- **mao-chiang-wisdom-skills**（GitHub 开源项目）：展示了如何把一位思想家的认知体系打包成 `.claude/skills/` 下的可安装 Skill，含辩论引擎和测试体系

对标分析后确定：巴菲特项目不做对抗式辩论，而是做**三层叠加演化**——L1 蒸馏原文 → L2 提取认知操作 → L3 开放框架边界给社区修正。

### 1.3 核心需求拆解

| 层 | 需求 | 对应的用户问题 |
|:--:|------|-------------|
| L1 | "巴菲特说过什么？" | 他的投资原则是什么？安全边际怎么定义？ |
| L2 | "巴菲特遇到没见过的事会怎么想？" | 面对 2008 年危机他会怎么做？一个 CEO 是否值得信任？ |
| L3 | "这个框架有什么盲区？谁可以修正它？" | 零利率环境下 DCF 怎么办？加密资产怎么纳入？ |

### 1.4 质量定义

参考 Perplexity 的 Eval 方法论和自身工程实践，设定可交付标准：

- L1 完成：核心概念卡片 ≥ 8 张，每条规则可追溯到原文出处
- L2 完成：决策流程总图可以对 2008 年金融危机产生合理推演
- L3 激活：盲区目录 ≥ 10 条，每条含可接手的 L3 种子问题
- 可发布：T0 触发 100% + T1 边界 100% + T2 框架 ≥ 80%

## 二、结构设计（2026-06-14 ~ 06-15）

### 2.1 三层架构

```
L1 蒸馏拟合："巴菲特说过什么？"
    ↓
L2 活化推演："巴菲特遇到没见过的事，会怎么想？"
    ↓
L3 开源演化："社区一起让这套框架自我修正"
```

三层不是先后顺序，是叠加深度。同一份素材在同一次阅读中，可以同时产出三个层级的产物。

### 2.2 目录结构设计

```
.claude/skills/buffett-wisdom/   ← 最终交付物
├── SKILL.md                     ← 主入口，AI 加载此文件
├── cognitive-framework/         ← L2 认知操作
├── evolution/                   ← L3 盲区 + 新场景 + 社区贡献
├── evals/                       ← T0/T1/T2 测试用例
├── scripts/                     ← 校验脚本
├── assets/templates/            ← 标准化模板
└── references/                  ← 素材索引

raw/buffett/                     ← 工作区（开发期中间产物）
├── research-books/              ← epub + 提取文本
├── notes/cards/                 ← L1 概念卡片
├── notes/cognition/             ← L2 认知操作 + 沿革考据
└── notes/                       ← 素材地图 + 方案 + 审计报告
```

设计原则：工作区和交付区分开。raw/ 是施工工地，.claude/skills/ 是交付楼盘。

### 2.3 分批计划：13 批次 × 三层穿透

项目被分解为 13 个独立批次，每批有明确的输入、任务和输出：

```
第一组（核心蒸馏）：批次 0-2
第二组（L2 深化合成）：批次 3-7
第三组（L3 激活 + 测试）：批次 8-9
第四组（产出合成 + 交付）：批次 10-13
```

### 2.4 工程方法论

在施工前即制定了完整的工程标准（`工程方法论.md`），包括：

- Agent 体系：Distiller → Cognitive Analyst → Red Team → Synthesizer
- 模型策略：Claude Opus 4 为首要目标，Sonnet/GPT 做回归
- 四阶段脚本策略：A（手动）→ B（judge.py）→ C（对抗验证）→ D（回归）
- 五层测试体系：T0 触发 → T1 边界 → T2 框架 → T3 跨模型 → T4 回归
- 每份产物的质量标准：必须包含什么、禁止包含什么

## 三、项目施工（2026-06-15 ~ 06-16）

### 3.1 批次 0：素材粗扫

**输入**：67 份原始文件
**任务**：分类、分级、去重、识别缺口
**输出**：`素材地图.md` —— 每份文件的年代/主题/类型/对应层级

### 3.2 批次 1：Cunningham 编译本 — 价值投资公理

**层级**：L1 + L2 并行
**任务**：读 epub 第 1-2 章，提取核心概念
**输出**：5 张 L1 卡片（价值投资/安全边际/内在价值/能力圈/市场先生）+ 1 份 L2 认知操作（好生意判断标准）+ 1 份双链图

### 3.3 批次 2：Cunningham 编译本 — 管理评估与资本配置

**层级**：L1 + L2 并行
**任务**：读 epub 第 1 章 B-G 节 + 第 5-7 章
**输出**：5 张 L1 卡片（护城河/管理层评估/资本配置/并购哲学/会计与税务）+ 2 份 L2 认知操作 + 1 份双链图

### 3.4 批次 3：沿革考据 — 四阶段认知演变

**层级**：L2 主导
**任务**：将批次 1-2 十大概念按巴菲特职业生涯四阶段（合伙基金→转型→成熟→晚期）重构时间线，标注每个概念的形成/修正节点
**输出**：沿革对照表 + 五个不连贯/矛盾之处分析

### 3.5 批次 4：认知操作流程合成

**层级**：L2 合成
**任务**：将批次 1-3 的零散认知片段整合为完整决策流程总图
**输出**：`decision-process-map.md` —— 第〇关信息过滤 → 四关排除 → 买入后监控 → 直觉 vs 流程分析 → 三级过滤统计

### 3.6 批次 5：表达风格 DNA 系统性提取

**层级**：L1 + L2
**任务**：从所有已读原文中提取句式结构（6 种）、标志性意象库（11+11）、语气确定性体系（5 级）、幽默模式（5 种）、禁忌与反模式
**输出**：`buffett-voice-dna.md`

### 3.7 批次 6：Hagstrom《巴菲特之道》交叉验证

**层级**：L1 + L2
**任务**：读 Hagstrom epub，将其 12 准则框架与我们的 10 概念体系逐条对照
**输出**：交叉验证表（一致项/补充项/差异项）+ L3 种子（3 个差异）

### 3.8 批次 7：芒格影响谱系

**层级**：L1 + L2
**任务**：读《穷查理宝典》，梳理芒格四维影响（吸收/重表达/缺失/分歧）
**输出**：`munger-influence.md` + 核心语录对照表

### 3.9 批次 8：盲区与边界体系化

**层级**：L2 → L3 过渡
**任务**：梳理巴菲特框架的边界——公开失误、未经历的市场环境、缺失维度、规模偏见、内部不自洽
**输出**：`blindspot-catalog.md` + 13 条 L3 种子

### 3.10 批次 9：2008 金融危机 L2+L3 实战测试

**层级**：L2 推演 + L3 审查
**任务**：用批次 4 决策流程推演 2007-2009 年五个关键节点，对照巴菲特实际行为
**输出**：`2008-crisis-war-game.md` —— 推演与实际一致度 A（优秀），一个遗漏（BNSF）、三个 L3 修正

### 3.11 批次 10：SKILL.md 初稿合成

**层级**：全层覆盖
**任务**：将批次 1-9 全部产出合成为单一 SKILL.md 文件
**输出**：`SKILL.md v1.0` —— 9 章节（角色/原则/流程/风格/边界/回答模式/示例/加载说明/元规则）

### 3.12 批次 11：测试用例编写

**任务**：按五层测试体系编写测试用例
**输出**：26 个用例（T0 触发 10 + T1 边界 8 + T2 框架 8）

### 3.13 批次 12：小批测试 → SKILL.md v2

**任务**：对照 26 个测试用例审查 SKILL.md，发现 2 处缺陷（触发词覆盖不足、边界声明缺失），修复后产出 v2

### 3.14 批次 13：审计交付

**任务**：文件迁移、目录搭建、审计报告、wiki 更新
**输出**：交付目录完整填充 + 26/26 测试全通过

### 3.15 v3.0 ~ v4.0 迭代（追加）

- v3.0：标准模式深度升级（纵向沿革 + 横向引证 + 历史锚点）+ `/crazy` 犀利模式
- v3.1：MCP 金融数据感知（有工具自动调，无工具温和提示）
- v4.0：7 大应用场景 × 结构化输出模板（对标毛泽东 Skill 的决策引擎设计）

### 3.16 工程化补齐（追加）

- 四份标准化模板（concept-card / cognition-record / blindspot-entry / test-case）
- 三份校验脚本（validate-evidence.py / validate-framework.py / validate-skill.py）
- 七份斜杠命令（使用侧 3 + 构建侧 4）
- 一份维护指南（README.md）
