# 思想家知识蒸馏：完整提示词工程

> 本文回答：如果要把一位思想家的认知体系蒸馏成 AI Skill，每一步怎么做，每一层的提示词怎么写，每一个质量门怎么设。

---

## 〇、总体流程

```
阶段0: 素材整备          ── 1-2h ── 素材地图
   ↓
阶段1: 概念蒸馏 (L1)     ── 4-6h ── 8-12张概念卡片
   ↓
阶段2: 边界界定           ── 2h   ── 盲区目录 + 行为边界
   ↓
阶段3: 沿革考据           ── 2h   ── 概念时间轴 + 思想来源对照
   ↓
阶段4: 思维框架合成 (L2)  ── 3h   ── 决策流程总图 + 场景模板
   ↓
阶段5: 表达风格提取        ── 2h   ── 句式/意象/语气DNA
   ↓
阶段6: SKILL.md 合成      ── 2h   ── 可安装Skill
   ↓
阶段7: 校验闭环           ── 2h   ── 测试用例 + 校验脚本 + 引用溯源
```

## 第一阶段：概念蒸馏 (L1)

### 1.1 概念卡片模板

这是整个蒸馏流程中最基础的单元。每张卡片 = 思想家的一个核心概念。

**提示词：蒸馏一张概念卡片**

```
你是一名知识蒸馏专家。从以下素材中提取 [思想家] 的一个核心概念。

## 素材
[粘贴原文段落]

## 任务
按以下五个字段生成一张概念卡片。每个字段都必须有原文依据，不允许编造。

### 原文出处
- 格式：> "原文直接引用"（出处：章节/年份/访谈名称）
- 要求：至少 2 条。引号中的内容必须和原文完全一致。

### 决策规则
- 格式：编号列表，每条规则可直接用于判断
- 要求：每条规则对应一条原文出处。用"必须""不""只"等可操作语言。

### 应用示例
- 格式：正面 ≥ 2 个，反面 ≥ 1 个
- 要求：每个示例包含：具体案例/年代/关键数据/规则如何应用/结果

### 边界条件
- 说明什么情况下此规则不适用
- 要求：至少 1 条。必须是具体的场景，不能笼统说"有时不适用"。

### 双链关联
- 列出与此概念相关的其他概念
- 用符号标注关系：→ 依赖 / ← 被需要 / ⇄ 相互定义
```

### 1.2 多卡片提示词（一次炼一章）

```
从以下文本中提取所有可识别的核心概念。每个概念独立成卡片。

## 素材
[粘贴一整章]

## 提取标准
- 筛选：只提取 [思想家] 反复使用的、有明确定义的概念
- 排除：一次性提及的、没有展开论述的观点
- 每个概念的卡片按模板格式输出

## 模板
[插入概念卡片模板]
```

### 1.3 质量检查

卡片提交后，用三个问题自检：

```
1. 每条决策规则都能指向一句原文吗？如果删掉出处字段，回答"是"。
2. 反面示例包含具体年代和数据吗？如果只有"他曾犯过错"，回答"不够"。
3. 边界条件说了什么情况下不能用吗？如果只有"需要谨慎"，回答"不够"。
```

## 第二阶段：边界界定

边界回答两个问题：**什么不能说**和**什么说了没用**。

### 2.1 行为边界（风险管理）

行为边界是硬约束——AI 绝对不能做的事。

**提示词：定义行为边界**

```
[思想家] 在以下场景中是如何划定自己的行为边界的？

1. 他从不做什么？（列出具体行为，每条附原文支撑）
2. 如果有人要求他做边界外的事，他会怎么回应？（附真实对话或文字记录）
3. 他的边界有没有被他自己的行为打破过？（找自相矛盾的例子）

输出格式：
- 边界： [具体行为] → 支撑： [原文/案例]
- 矛盾： [他什么时候破了这个边界] → 解释： [他自己怎么说？]
```

### 2.2 知识盲区（框架边界）

知识盲区是软边界——框架在这个领域可能不准。

**提示词：识别知识盲区**

```
找出 [思想家] 的框架在以下环境中可能失效的场景：

1. 他从未经历过的市场/社会环境（如零利率、量化交易、加密资产）
2. 他的框架中系统性缺失的维度（如 ESG、地缘政治、供应链）
3. 他的个人条件带来的偏见（如规模、时代、地理位置）
4. 他公开承认过但框架未能解释的矛盾

每个盲区输出为一个 L3 种子：
- 场景描述
- 框架为何失效
- 一个可被社区接手的具体研究问题
```

### 2.3 确定性分层

**提示词：提取确定性体系**

```
阅读 [思想家] 的著作，标注他如何表达不同程度的确定性：

- L5 绝对：他用什么词/句式表达铁律？（如"绝不""必须""任何…乘以零都等于零"）
- L4 极高：他用什么词/句式表达高度确信？（如"我相信""几乎可以肯定"）
- L3 默认：他的中性/默认表达方式是什么？
- L2 推测：他极少用的推测性语言是什么？
- L1 承认无知：他用什么方式说"我不知道"？

输出一个确定性→行动映射表：
L5 说了什么 → 他做了什么（不可以商量的事）
L1 说了什么 → 在什么场景下说的
```

## 第三阶段：沿革考据

概念的真正含义往往藏在它的**演变史**里，不在它今天的定义里。

### 3.1 概念的时间轴重构

**提示词：追踪一个概念的演变**

```
追踪 [概念名] 在 [思想家] 职业生涯中的演变。按以下阶段逐段提取：

阶段1（[年代范围]）：这个概念刚出现时是什么样的？谁影响了他？
阶段2（[年代范围]）：什么事件/人改变了他的理解？定义发生了什么变化？
阶段3（[年代范围]）：概念成熟定型后是什么样？
阶段4（[年代-今]）：这个概念在他晚年有没有再被修正？

输出格式：
| 阶段 | 年代 | 定义 | 修正原因 | 支撑原句 |
|------|------|------|---------|---------|
```

### 3.2 思想来源对照

**提示词：对照原始出处**

```
[思想家] 声称他某个概念来自 [导师A]。请做以下对照：

1. 找到 [导师A] 对这个概念的原始定义（原文引用）
2. 找到 [思想家] 对这个概念的当前定义（原文引用）
3. 逐条对比差异
4. 判断：这属于"忠实传承"还是"保留标签内容替换"？

输出一张对照表：
| 维度 | 导师定义 | 思想家的定义 | 是否改变 |
```

**常见陷阱**：思想家倾向于**高估导师贡献**以保持敬意，同时**低调地替换了概念内容**。不读导师原著就发现不了这个偏差。

### 3.3 多人影响谱系

**提示词：构建影响谱系**

```
[思想家] 提到了多位影响者（[导师1]、[导师2]、[搭档1]）。

对每条概念标注它的思想来源：
- 完全来自某人的 → 标来源
- 来自某人但被重新表达的 → 标"吸收但重表达"并描述变化
- 自创的 → 标"原创"
- 某人强调但思想家体系中没有的 → 标"缺失"

输出四维分类：
1. 完全吸收（忠实传承）
2. 转化吸收（标签不变内容变）
3. 未被吸收（另一个人的思想，他没有采纳）
4. 公开分歧
```

## 第四阶段：思维框架合成 (L2)

L1 告诉你"他有什么概念"。L2 告诉你"他怎么用这些概念做决策"。

### 4.1 决策流程提取

**提示词：提取决策流程**

```
基于我们已有的所有概念卡片，推导 [思想家] 面对 [典型场景] 的决策流程。

要求：
1. 使用 ASCII 决策树格式（├─ 第N关 → └─ 判断）
2. 每一步必须有至少一张概念卡片支撑（标注卡片名称）
3. 每一步必须标注判断通过/不通过的标准
4. 每一步必须给出不通过时的出口（放弃/等待/重新评估）
5. 区分"自动化判断"（秒级/条件反射）和"深度分析"（需要时间/数据）

输出格式：
输入：[场景]
  ├─ 第1关名：判断标准 → 不通过→出口
  ├─ 第2关名：判断标准 → 不通过→出口
  └─ ...
```

### 4.2 排除模式识别

**提示词：识别排除模式**

```
[思想家] 的决策方式本质上是一个排除系统——通过不断排除不该做的事，剩下的才是该做的。

列出他所有的"秒级排除"（一个信号就否决的模式）：
- 排除模式1：[具体信号] → 原文中的典型回应
- 排除模式2：...

估算：这些排除模式加起来覆盖了多少%的日常"机会"？
```

### 4.3 场景→输出模板

**提示词：设计场景模板**

```
基于决策流程，为 [思想家] 的典型使用场景设计输出模板。

每个场景包含：
- 触发信号（用户说什么时匹配这个场景）
- 输出格式（固定几段，每段是什么）
- 底线（这个场景下绝不能做什么）
- 场景冲突时的优先级（用户一句话触发多个场景时用哪个）

场景数量控制在 5-8 个。标准：每个场景必须有独立的输出模板——和另一个场景格式过于接近就应该合并。
```

### 4.4 实战推演验证

**提示词：用历史事件验证框架**

```
用我们建立的决策流程推演 [思想家] 在 [历史事件] 每个关键节点的判断。

节点1（[日期]）：[场景描述]
  → 框架推演：他会怎么判断？
节点2（[日期]）：[场景描述]
  → 框架推演：他会怎么判断？

然后对照实际历史记录：
- 推演与实际一致之处 → 框架置信度高
- 推演与实际不一致之处 → 标记为"需要修正"
- 推演遗漏之处 → 标记为"框架盲区"
```

## 第五阶段：表达风格提取

### 5.1 句式结构

**提示词：提取标志性句式**

```
阅读 [思想家] 的原文，找出他反复使用的句式结构。每种句式给一个名称和至少三个实例。

关注：
- 反转句式（常识→但是→他相反的观点）
- 反问+自答（他问一个问题→自己回答）
- 朴素比喻（用日常生活解释复杂概念）
- 否认排比（列举不做什么）
- 引用权威后反驳（某人说过X→但实际Y）

输出格式：
句式名：[名字]
模式： [句式模板]
实例1/2/3： [原文]
使用频率： [高/中/低]
```

### 5.2 意象库

**提示词：建立意象库**

```
列出 [思想家] 反复使用的比喻和意象。按使用频率排序。

输出分两类：
- 核心意象（贯穿整个职业生涯，反复使用）：[名字]→[代表含义]→[首次出现年代]
- 辅助意象（偶尔使用，增强生动性）：[名字]→[出处]
```

### 5.3 确定性体系

**提示词：提取确定性分层**

```
阅读 [思想家] 的原文，识别他如何标记不同级别确定性。

找出以下各级别的典型句式：
- 绝对（从不/必须/不可能）：用于什么主题？
- 极高（我相信/几乎肯定）：用于什么主题？
- 默认（我认为/我们倾向于）：频率？
- 推测（也许/可能）：他什么时候用？
- 不知（我不知道/无法判断）：他说"不知道"时有什么模式？

输出五级表 + 每一级映射到的主题类型。
```

### 5.4 双重模态：书面 vs 即席

**提示词：区分书面和口语风格**

```
对比 [思想家] 在书面（著作/年报）和即席（访谈/演讲）中的表达差异：

| 维度 | 书面表达 | 即席表达 | 转换规律 |
|------|---------|---------|---------|
| 句子长度 | | | |
| 确定性 | | | |
| 幽默程度 | | | |
| 引用频率 | | | |

关键问题：他书面的核心概念在口语中被压缩成了什么样的简短表述？
```

## 第六阶段：SKILL.md 合成

### 6.1 角色定义

```
你是 [思想家]，[身份描述]。你从 [起始年代] 开始 [主要工作]，经历了 [关键事件]。
你的思维方式不是天生的——它从 [早期影响] 开始，经过 [中期转变]，最终形成了现在的体系。

你的核心身份特征：
- [特征1]：[一句话描述]
- [特征2]：[一句话描述]
- [特征3]：[一句话描述]
```

### 6.2 核心原则

每一条原则 = 一句原话引用 + 三条可执行规则。

### 6.3 决策框架

从第四阶段的决策树精简而来——把完整的 ASCII 树压缩为 SKILL.md 中可读的四段。

### 6.4 表达风格

从第五阶段提取：句式规则 + 意象清单 + 确定性体系 + 禁止语言列表。

### 6.5 边界

从第二阶段提取：回避领域 + 已知局限 + MCP 感知规则（如有）。

### 6.6 场景模板

从第四阶段提取：7 个场景 × 触发信号 × 输出格式 × 底线。

## 第七阶段：校验闭环

### 验证脚本体系

```
validate-evidence.py    ← L1 卡片形式合规
validate-framework.py   ← L2 框架素材支撑
validate-skill.py       ← SKILL.md 用例通过率
factcheck.py            ← 引用内容真实性
```

### 测试用例三层

```
T0 触发测试（正向触发/负向不触发）
T1 边界测试（不推荐/不预测/不攻击/不承诺）
T2 框架测试（决策流程正确/意象使用/人称一致/确定性分层）
```

## 完整提示词链（一次跑通全流程）

如果你需要一次性启动整个蒸馏流程，用这个链式提示词序列：

```
提示词 1（素材粗扫）:
  通读以下素材列表，输出素材地图：
  [素材清单]
  每份标注：年代/类型/预估可提取的概念/对应层级

提示词 2（概念蒸馏）:
  基于素材地图，从 [核心素材] 中提取概念卡片。
  使用概念卡片模板。每张卡片五个字段不能少。

提示词 3（边界界定）:
  基于已提取的概念卡片，找出框架的边界和盲区。
  行为边界 + 知识盲区 + 确定性分层。

提示词 4（沿革考据）:
  追踪已提取概念的时间演变和思想来源。
  概念时间轴 + 导师对照 + 影响谱系。

提示词 5（思维框架）:
  基于概念卡片和决策模式，合成完整决策流程。
  决策树 + 排除模式 + 场景模板。

提示词 6（实战验证）:
  用 [历史事件] 测试决策流程。对比推演和实际。

提示词 7（表达风格）:
  提取句式、意象、确定性体系、书面/口语差异。

提示词 8（SKILL.md 合成）:
  把前面所有产出合成为 SKILL.md。
  角色+原则+流程+风格+边界+场景。
```
